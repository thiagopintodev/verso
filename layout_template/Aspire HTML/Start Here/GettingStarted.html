-- ORGANIZATION:
First - a note on the organization of this template. I have placed all essential non-html files inside of the "assets" folder. There you'll find the primary images, javascript, css, and other files that are essential to each version of this template.

Above this folder I have grouped each "color" inside of it's own folder. You'll notice that there are 7 main .html files and a contact form .php file. You will notice that each of these files is referencing it's respecitive "color style" stylesheet at the very top of the header - ie: the white version references white.css. The blue version regerences /colors/blue/blue.css .

The files inside of each theme folder are just for quick access - you can re-organize their proximity to the files in the "assets" folder at your own pleasure.

The CSS file in those color folders (black.css, blue.css, etc.) doesn't cover the main styling for the template, only those styles that differentiate each theme from each other.

A) HTML Structure This theme is a fixed layout with two possible configurations: Full Width, and 2 Column.
All of the information within the main content area is nested within a div with an id of "container". The header section is located under the "header" class.

The navigation is located in the "sf-menu"" class inside the "header".
The slider at the top of the homepage is located inside the "slider" id.
The homepage 4 blurbs of information are nested inside the div with the class of "quick-links".
The left column content is within a div with an id of "page".
The sidebar's (column #2) content is within a div with an id of "sidebar".
The footer content is within a div with an id of "footer".

If you would like to edit the color, font, or style of any elements in one of these columns, you would do the following:

#content a {<br /> color: #someColor;<br /> }
If you find that your new style is not overriding, it is most likely because of a specificity problem. Scroll down in your CSS file and make sure that there isn't a similar style that has more weight.

I.E.
.green a {<br /> color: #someColor;<br /> }
So, to ensure that your new styles are applied, make sure that they carry enough "weight" and that there isn't a style lower in the CSS file that is being applied after yours.

The styling for this theme are stored in 3 key locations: core.css (main styling) , quickstyles.css , and finally, there are a handful of other CSS files that include styling for the various third party plugins, such as the dropdown, prettyPhoto, etc. I have taken the time to separate the most important styling components (the background graphic and the main link colors) into their own CSS files, located in the "css/quick-styles" folder.
B) CSS I'm using one main CSS file in this theme to load all other stylesheets (core.css). There several support stylesheets (for IE browsers and the jQuery stuff). The site also includes a generic reset stylesheet (reset.css) inside the hacks folder. Many browser interpret the default behavior of html elements differently. By using a general reset CSS rule, we can work round this. This file also contains some general styling, such as anchor tag colors, font-sizes, etc. Keep in mind, that these values might be overridden somewhere else in the file.

The main file (core.css) contains all of the universal stylings for the pages. The file is separated into sections using:

/* Main Section */
some code

/* Footer */
some code
etc, etc.

If you would like to edit a specific section of the site, simply find the appropriate label in the CSS file, and then scroll down until you find the appropriate style that needs to be edited.

I've also done by best to keep this code as organized as possible. Take a few minutes to familiarize yourself with what's in each file before you start your styling, and it should be a breeze.

2) losangeles.js | The main JS file that runs any necessary activation scripts for the other files.
3) jquery-1.4.2.min.js | The main jQuery script.
4) jquery.prettyPhoto.js | The jQuery script that drives the image lightbox.
5) jquery.nivo.slider.pack.js | The jQuery script that drives the image slider on the homepage.
6) superfish.js | The dropdown menu script.
7) hoverIntent,js | A support file for the dropdown menu.
8) cufon-yui.js | The main text-replacement script.
9) avenir.js | The "font" file for the CUFON font replacement.
10) DD_belatedPNG_0.0.8a-min.js | A PNG fix script.

F) CUFON Implementation This site uses CUFON for font-replacement in the navigation, the headlines, and several other spots. It's accomplished using the basic CUFON-installation as described on their site. Notice the references to CUFON fonts inside the header-section of each HTML file, and the "replacement" declaration inside the losangeles.js file.

The fonts are then given their style inside the main template stylesheets (most are in core.css)

You can change the color of some of the CUFON fonts from the color specific stylesheet - ie: /assets/colors/white/white.css

If you need to change to another font because of language or other special character issues - just pick the font that you want to use, take it to http://cufon.shoqolate.com/generate/ - and generate a new font. Then reference the new font file inside the load.js file and the aspire.js file.
G) Customization and Contact Form Usage You can quickly create your own custom stylesheets by over-writing the color specific CSS files with your own adjustments. Keep in mind that you will need to keep track of the original styling in case your new styles compete with them. You can also add new background graphics and other customizations here as well.

The Contact Form is quite simple to use, but as it's the only PHP file, I should walk you through it. It's been built using the Super Simple Contact Form Maker : http://supersimple.org/contact. First, there's a good chance that you'll want to ignore the large block of PHP at the top of this page. If all you want to do is setup your email address, just scroll down to line 230-233 in the code:
//now get the recipient(s) $_to[] = "YourEmail@Address.here";
Just enter your email address instead of "YourEmail@Address.here" and the form will be setup. To make customizations to this form, I highly recommend reading the documentation at the Super Simple Contact Form website, as that's what I used to create the form. Just insert the code that they give you into this framework and you'll be good to go.
H) Homepage Slider Usage You'll locate the default "NivoSlider" inside the main index.html file inside the "slider" section of code:

<!-- Slider --> <div id="slider"> <a href="#"><img src="../assets/img/slider/example01.jpg" alt="" /></a> <img src="../assets/img/slider/example02.jpg" alt="" /> <a rel="prettyPhoto[gallery]" href="#"><img src="../assets/img/slider/example03.jpg" alt="" title="This is an example of a caption! Add your own custom title and description here." /></a> <a rel="prettyPhoto[gallery]" href="#"><img src="../assets/img/slider/example04.jpg" alt="" /></a> <a rel="prettyPhoto[gallery]" href="#"><img src="../assets/img/slider/example05.jpg" alt="" /></a> </div> <!-- </Slider -->
The code here is quite simple - just add a new IMG element inside the "slider" DIV for each new slide that you want. You can wrap the image tag with a Hyperlink tag to make it link to another page. You can edit the destination of the slide from the HREF item, and the source of the image from the SRC item. You will notice that by adding a TITLE="..." to any of your image tags will trigger the slider to show a "caption" underneath the image.

If you want to launch a lightbox, just insert the [rel="prettyPhoto[YourGalleryTitle]"] into the hyperlink tag. Read more about how to use the lightbox at the PrettyPhoto website.
